## 1. 主要功能模块

看板基于 React + Ant Design 构建，主要由以下模块组成：

| 模块 | 位置 | 功能说明 |
|------|------|----------|
| **顶部状态栏** | 页面顶部 | 显示项目名称、当前阶段、WebSocket 连接状态（绿点=已连接） |
| **Agent 状态总览** | 左侧主区域 | 展示 16 个 Agent（Orchestrator / PM / 后端 / 前端 / 测试 / 评审 / 部署等），每张卡片显示图标、名称、状态（idle/busy/error）、当前任务、进度 |
| **项目概览** | 右侧上半部分 | 当前 Phase、Phase 进度条、任务完成数/总数、Token 消耗量、四个门禁状态（Gate 1~4） |
| **实时操作日志** | 右侧下半部分 | 滚动展示系统行为日志，支持按等级（INFO/WARN/ERROR）和关键字筛选 |
| **底部分页/统计** | 页面底部 | 日志总数、当前页码、自动刷新状态 |

## 3. 数据录入与管理

### 3.1 Agent 数据（自动同步）
Agent 状态由后端调度器（`scheduler`）每 2 秒采集一次，**前端无需手动录入**。如需手动添加新 Agent：
1. 进入后端目录：`g:\ai-gongju\Loop-agent\mcp-monitor-dashboard\backend`
2. 编辑 `app\models\agent.py`，在默认列表中追加新 Agent 配置
3. 重启后端服务

### 3.2 项目数据
项目概览数据由后端根据"当前运行的 Loop-Harness-Agent 项目"自动同步。手动方式：
- 修改 `app\models\project.py` 中的 `DEFAULT_PROJECT`
- 通过 REST API `POST /api/v1/project/update` 更新

### 3.3 日志数据
日志由系统行为自动写入 `logs` 表，可在 Settings → Clear Logs 清空展示。

## 5. 权限设置流程

当前版本（v1.0）默认**单管理员模式**。多权限方案规划中：

1. **临时方案**：通过环境变量 `MONITOR_AUTH_TOKEN` 控制访问
   ```bash
   # 后端启动时
   set MONITOR_AUTH_TOKEN=your-secret-token
   uvicorn app.main:app --host 0.0.0.0 --port 8000
   ```
2. **正式方案（规划）**：基于 RBAC 的三角色体系
   - **Admin**：可操作（删除、暂停、重启 Agent）
   - **Engineer**：可读可写日志评论
   - **Viewer**：只读

## 6. 常见操作场景演示

### 场景 A：实时观察一个任务从开始到完成

1. 打开 Dashboard（`http://localhost:3000`）
2. 关注"项目概览"区域的"当前阶段"
3. 观察"Agent 状态总览"：对应的 Agent 卡片会从 `idle` 变为 `busy`，进度条开始递增
4. 完成后，Agent 状态变回 `idle`，对应日志出现在"实时操作日志"中
5. 项目概览的"任务完成数" +1

### 场景 B：排查 WebSocket 断连问题

1. 顶部状态栏的连接点变红（断开）
2. 进入"实时操作日志"，筛选 `WARN` 等级
3. 搜索关键字 `disconnected` 或 `reconnect`
4. 查看断连时间戳和原因（code 1006 = 异常断开）
5. 等待 1-30 秒指数退避重连，或刷新页面强制重建连接

### 场景 C：导出日志给同事

1. 进入"实时操作日志"
2. 设置筛选条件（时间范围 + 等级）
3. 点击 📥 导出按钮
4. 浏览器下载 `monitor_logs_YYYYMMDD_HHMMSS.csv`
5. 通过企业微信/邮件发送

### 场景 D：检查 4 个门禁的通过情况

1. 看"项目概览"中 4 个门禁徽章（Gate 1~4）
2. 绿色 = 通过 ✅，黄色 = 进行中 🟡，红色 = 失败 ❌
3. 点击任一门禁 → 进入"门禁评审页"查看详情
4. 失败的 Gate 会在"实时操作日志"中产生 ERROR 日志

## 7. 使用注意事项

### 7.1 浏览器兼容性
- ✅ Chrome / Edge 100+
- ✅ Firefox 95+
- ⚠ Safari 15+ 部分动画可能掉帧
- ❌ IE 不支持

### 7.2 网络要求
- 前端 → 后端 API：`http://localhost:8000/api/v1/*`（生产环境使用 HTTPS）
- 前端 → 后端 WebSocket：`ws://localhost:8000/ws`（生产环境使用 WSS）
- **如果看到 `ERR_CONNECTION_REFUSED`**：检查后端是否启动（`netstat -ano | findstr :8000`）

### 7.3 数据安全
- 不要在前端硬编码任何密钥
- 生产环境必须配置 CORS 白名单
- WebSocket 不传输敏感信息（仅状态、进度、日志）
- 关闭浏览器前点击"安全退出"，清除本地缓存

### 7.4 性能建议
- 同时打开看板的客户端不超过 5 个（WebSocket 连接数限制）
- 日志超过 1000 条会自动分页（默认每页 100 条）
- 长时间不操作请点"暂停推送"，避免浏览器卡顿

### 7.5 常见报错处理

| 错误现象 | 原因 | 解决方案 |
|----------|------|----------|
| 顶部连接点一直红 | WebSocket 未连上 | 检查后端是否启动；检查 Vite 代理配置 |
| 日志显示 500 | 后端内部错误 | 查看后端日志；联系 @Bug-Defect-Repairer |
| 页面空白 | 前端 JS 报错 | F12 看 Console；刷新页面 |
| 代理 ECONNRESET | Vite 代理转发失败 | 重启 Vite；检查后端 8000 端口 |
