## 4. 🛡️ 4 道门禁:问题产品休想上线!

**图片文件**: `04-gates.png`

**Hook(首行)**: AI 写的代码不测试就上线?等着 P0 事故吧

**正文**:

质量不是测出来的,是流程里长出来的!

Loop Agent 的 4 道门禁,每道只盯一个维度:

🚧 Gate 1 · 代码审查
   负责人:@Code-Reviewer
   标准:0 Blocker + 0 Major
   不通过 → 回到 Phase 5 重写

⚡ Gate 2 · 性能压测
   负责人:@Performance-Engineer
   标准:P95 ≤ 300ms,错误率 ≤ 0.1%
   不通过 → 回去优化

🧪 Gate 3 · 功能测试
   负责人:@全栈测试员
   标准:P0/P1 Bug = 0,P2 ≤ 3 个
   不通过 → 回去修

👑 Gate 4 · 终审
   负责人:@Final-Reviewer
   标准:风险等级 ≤ LOW
   不通过 → 全流程返工

## 6. 🔌 MCP Server · AI 时代的 USB 标准

**图片文件**: `06-mcp-server.png`

**Hook(首行)**: 换 AI 模型就要重学一遍?这事儿被 MCP 终结了

**正文**:

MCP = Model Context Protocol,AI 世界的 USB 接口!

🤔 你可能听过这个痛点:
今天用 Claude 写了一套 AI 工作流,
明天想换 GPT,得重新写一遍调用代码 😭

✨ Loop Agent 的解法:把调度逻辑全部包成 MCP Server!

🎯 6 个标准动作,所有 AI 都能调:
• start_loop 启动流水线
• get_status 查进度
• spawn_agent 派活给某个 AI 角色
• save_blackboard 写黑板
• list_agents 查花名册
• abort_loop 紧急刹车

🔓 好处:
• 换 AI 不换流程(Claude→GPT→Gemini 都行)
• 规矩由 Server 守,AI 不会偷懒或忘事
• 多人多 AI 并行使用同一个 Server

💡 产品经理启示:
你的产品要不要 Agent 化?看有没有可被一句话描述的明确动作!
