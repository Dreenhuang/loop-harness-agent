## 一、用户研究摘要

### 1.1 目标用户特征

| 维度 | 项目负责人 (Primary) | 运维工程师 (Secondary) | 技术负责人 (Tertiary) |
|------|---------------------|----------------------|----------------------|
| **核心诉求** | 全局掌控 | 故障快速响应 | 深入分析 |
| **使用频率** | 工作日高频 | 7×24 待命 | 调试时密集 |
| **耐心程度** | 低（要快） | 中（要准） | 高（要细） |
| **技术背景** | 产品/管理 | 运维/DevOps | 开发/架构 |
| **典型场景** | 晨会看进度、周会汇报 | 收到告警后排查 | 调试某个 Agent |

### 1.2 用户痛点映射

| 痛点 | 严重度 | 频率 | 本产品解决方案 |
|------|--------|------|---------------|
| Agent 状态不透明，不知道在干什么 | 🔴 高 | 每日多次 | F001: 实时状态矩阵 |
| 日志分散难查找，问题定位慢 | 🔴 高 | 出问题时 | F002: 统一日志流 + 搜索 |
| 进度无法量化，难以预估完成时间 | 🟡 中 | 每日 | F003: 进度条可视化 |
| 需命令行操作 MCP Server | 🟡 中 | 偶尔 | F005: Web 控制面板 |
| 多个指标需手动汇总 | 🟢 低 | 周报时 | F004: 自动化仪表盘 |

### 3.2 页面区域详细说明

#### 区域 A: 顶部控制栏 (Header)
**位置**: 页面最顶部，固定高度 56px
**内容**:
- 左侧：Logo + "MCP Monitor Dashboard" 标题
- 右侧：
  - 系统状态指示灯（绿色圆点=运行中，灰色=已停止，红色=异常）
  - 三个控制按钮：【启动】(绿) 【停止】(红) 【重启】(黄)
  - 设置图标按钮（M2 功能，M1 灰色禁用态）

**交互规则**:
- 按钮 hover 时显示 tooltip 描述
- 点击【停止】【重启】弹出确认对话框
- 操作进行中按钮显示 loading spinner + disabled 态
- 操作完成后显示 toast 提示（成功绿色/失败红色，3秒自动消失）

---

#### 区域 B: Agent 状态矩阵 (左上)
**位置**: 左上象限，占宽度 50%，高度 45%
**内容**: 4×4 网格排列的 16 个 Agent 卡片

**卡片内部结构**:
```
┌─────────────────┐
│  [图标] 名称     │  ← 第一行：图标 + 显示名
│  [状态标签]      │  ← 第二行：状态 badge
│  进度: ████░ 67% │  ← 第三行：进度条（仅 running 状态显示）
│  任务: xxxxx     │  ← 第四行：当前任务描述（截断）
└─────────────────┘
```

**状态视觉编码**:
| 状态 | 边框颜色 | 背景色 | 图标 | 文字 |
|------|---------|--------|------|------|
| `idle` | #D9D9D9 灰 | #FAFAFA | ⚪ 灰圈 | "空闲" |
| `running` | #1890FF 蓝 | #E6F7FF 浅蓝 | 🔵 蓝圈 | "运行中" |
| `error` | #FF4D4F 红 | #FFF2F0 浅红 | 🔴 红圈 | "异常" |
| `complete` | #52C41A 绿 | #F6FFED 浅绿 | 🟢 绿圈 | "已完成" |

**交互行为**:
- Hover: 卡片轻微上浮（translateY -2px）+ 阴影加深
- Click: 高亮选中态（蓝色边框加粗），M1 无跳转，M2 跳转详情页
- Error 状态卡片：边框脉冲动画（2秒循环），吸引注意力

**筛选功能**:
- 矩阵上方横向 Tab 栏：`全部` | `运行中 (3)` | `空闲 (12)` | `异常 (1)` | `已完成 (0)`
- 点击 Tab 即刻过滤，带过渡动画（200ms fade）
- 数字动态更新（如某 Agent 从 running → complete，数字自动变化）

**位置**: 右上象限，占宽度 50%，高度 45%
**内容**: 关键指标卡片网格（2列 × 3行）

**指标卡片列表**:

| 卡片 | 图标 | 数据格式 | 颜色主题 |

**交互行为**:

#### 区域 D: 操作日志流 (左下)
**位置**: 左下象限，占宽度 55%，高度 50%
**内容**: 时间序日志列表（虚拟滚动）

**单条日志结构**:
```
[14:32:15.123] [Architect] [INFO ] 架构设计文档已保存到 docs/mcp-monitor-dashboard/Architecture.md
```

**视觉层次**:
- 时间戳：等宽字体（Monaco/Courier），灰色 #8C8C8C，字号 12px
- Agent 名：加粗，可点击（点击过滤仅显示该 Agent 日志）
- 级别 Badge：
  - INFO: 默认文字
  - WARN: 黄底黑字 (#FFF7E6 背景)
  - ERROR: 红底白字 (#FFF1F0 背景) + 左侧红色竖线
  - DEBUG: 灰色小字
- 消息正文：正常字体，可换行

**工具栏**（日志区域顶部）:
- 搜索框：输入关键词实时过滤（debounce 300ms）
- 筛选下拉：按级别多选（Checkbox）
- Agent 下拉：按 Agent 多选
- 暂停/播放按钮：控制自动滚动
- 清空按钮：清空当前过滤结果
- 导出按钮：导出为 TXT/JSON（M2 功能）

**滚动行为**:
- 新日志到达时：如果用户在底部（距底部 < 100px），自动滚到底部
- 用户手动向上滚动时：暂停自动滚动，右下角浮动按钮 "▼ 有 N 条新日志"
- 点击浮动按钮：快速滚到底部并恢复自动滚动

## 四、关键交互流程

### 4.1 首次加载流程

```
用户打开 http://localhost:3000
  ↓
[0.0s] 显示 Loading Spinner（品牌色旋转动画）
  ↓
[0.5s] 骨架屏加载（4个区域的灰色占位块）
  ↓
[1.0s] WebSocket 连接建立
  ↓
[1.2s] 收到 init 消息（全量数据）
  ↓
[1.5s] 渲染实际内容（Agent 矩阵 + 仪表盘 + 日志 + 进度）
  ↓
[1.8s] 骨架屏淡出，实际内容淡入（crossfade 300ms）
  ↓
[2.0s] 首屏渲染完成 ✓（满足 ≤ 3秒要求）
  ↓
进入消息循环：每 2 秒接收增量更新
```

### 4.2 Agent 状态变化流程

```
Backend 检测到 Architect 从 idle → running
  ↓
DataCollector.collect() 发现状态变化
  ↓
Broadcaster.broadcast({
  type: 'agent_status_update',
  data: { id: 'architect', status: 'running', ... }
})
  ↓
前端 WebSocket.onmessage 接收
  ↓
useWebSocket Hook 解析消息
  ↓
agentStore.updateAgentStatus(architect, newData)
  ↓
Zustand 触发 re-render
  ↓
对应 AgentCard 组件更新：
  - 状态 badge: ⚪idle → 🔵running
  - 背景色: 灰 → 浅蓝
  - 进度条: 隐藏 → 显示 0%
  - 动画: 200ms transition
```

### 4.3 日志搜索流程

```
用户在搜索框输入 "error"
  ↓
[debounce 300ms 等待]
  ↓
触发 API 调用: GET /api/v1/logs?keyword=error
  ↓
后端查询 SQLite FTS5 索引
  ↓
返回匹配结果（< 200ms）
  ↓
前端更新日志列表：
  - 仅显示匹配的日志条目
  - 匹配关键词高亮（黄色背景标记）
  - 结果计数显示："找到 23 条匹配"
  - 列表平滑过渡（fade 150ms）
```

### 4.4 MCP Server 控制流程

```
用户点击【停止】按钮
  ↓
弹出确认对话框：
  ┌─────────────────────────────┐
  │  ⚠️ 确认停止                 │
  │                             │
  │  即将停止 MCP Server，所有    │
  │  正在运行的 Agent 将中断。    │
  │                             │
  │  是否继续？                   │
  │                             │
  │        [取消]  [确认停止]    │
  └─────────────────────────────┘
  ↓
用户点击【确认停止】
  ↓
按钮变为 loading 态（spinner + "正在停止..."）
  ↓
POST /api/v1/system/stop
  ↓
后端 ProcessManager.stop()
  ↓
子进程 terminate() + 等待 5 秒
  ↓
返回 success response
  ↓
前端接收响应：
  - Toast 提示："MCP Server 已停止"（绿色，3秒）
  - 按钮恢复常态
  - 状态指示灯变灰
  - 所有 Agent 卡片变为 ⚪ idle 或 ⚠️ disconnected
  - 日志流追加一条系统日志 "[System] MCP Server stopped at ..."
```

## 六、无障碍访问 (Accessibility)

### 6.1 键盘导航
- Tab 键顺序：Header → 控制按钮 → Agent 卡片（左→右，上→下）→ 搜索框 → 日志列表 → Footer
- Enter/Space：激活按钮、展开卡片
- Escape：关闭弹窗/模态框、取消操作
- 方向键：在 Agent 矩阵内移动焦点

### 6.2 屏幕阅读器支持
- 所有图标配备 aria-label
- 状态变化通过 aria-live="polite" 通知
- 进度条使用 role="progressbar" + aria-valuenow
- 表格数据使用语义化 HTML（table/th/td）

### 6.3 色彩对比度
- 正文文字对比度 ≥ 4.5:1 (AA 标准)
- 大标题对比度 ≥ 3:1 (AA 标准)
- 状态颜色不作为唯一区分手段（配合图标/文字）

## 七、错误状态设计

### 7.1 WebSocket 断连
```
┌─────────────────────────────────────────────┐
│                                             │
│          📡 连接中断                         │
│                                             │
│  正在尝试重新连接... (第 3 次)               │
│  预计恢复时间: 约 4 秒                       │
│                                             │
│  [手动重连]                                  │
│                                             │
└─────────────────────────────────────────────┘
```
- 位置：页面中央模态层（overlay）
- 背景：半透明黑色遮罩（opacity 0.5）
- 自动消失：重连成功后自动关闭
- 手动重连：点击按钮立即尝试

### 7.2 数据加载失败
```
┌─────────────────────────────────────────────┐
│                                             │
│          ❌ 数据加载失败                     │
│                                             │
│  无法获取 Agent 状态数据                    │
│  错误代码: 500 INTERNAL SERVER ERROR        │
│                                             │
```
