## 分支管理规范

### 分支命名
- 开发分支: `feature/mcp-monitor-dashboard`
- 主分支: `master`
- 发布分支: `release/mcp-monitor-v*.*.*`
- 热修复分支: `hotfix/mcp-monitor-*`

### 同步策略
1. **同步频率**: 每2个工作日必须与主分支同步一次
2. **同步方式**: 使用 rebase 方式，保持线性历史
3. **冲突处理**: 遇到冲突立即停止，通知总控协调解决

### 保护规则
- ✅ 禁止直接 push 到 master
- ✅ 必须通过 PR 合入
- ✅ 至少需要 1 名评审者审批
- ✅ CI/CD 检查必须全部通过

### 同步命令

```powershell
# PowerShell 同步脚本
.\scripts\sync-with-main.ps1
```

```bash
# Bash 同步脚本
./scripts/sync-with-main.sh
```

## 质量标准

### 性能指标
- 页面加载时间: ≤ 3 秒
- 数据更新频率: ≥ 2 秒/次 (WebSocket)
- 响应时间: ≤ 500ms (API)
- 并发用户数: ≥ 100

### 兼容性
- 桌面端: Chrome, Firefox, Edge, Safari (最新版)
- 移动端: iOS Safari, Android Chrome
- 响应式断点:
  - Mobile: < 768px
  - Tablet: 768px - 1024px
  - Desktop: > 1024px

### 技术栈约束
- 前端: React 18 + TypeScript 5 + Vite
- 后端: Node.js 20+ / Python 3.11+
- 实时通信: WebSocket (ws 库)
- 样式方案: Tailwind CSS 或 CSS Modules
- 构建工具: Vite / webpack
- 包管理器: pnpm / npm

## 工作流程

### 开发流程
1. 从 `feature/mcp-monitor-dashboard` 创建功能子分支
2. 完成开发后提交 PR 到 `feature/mcp-monitor-dashboard`
3. 通过代码审查后合并
4. 每 2 个工作日 rebase master

### 提交规范
使用 Conventional Commits 规范:
- feat: 新功能
- fix: Bug 修复
- docs: 文档更新
- style: 代码格式调整
- refactor: 重构
- test: 测试相关
- chore: 构建/工具相关

示例:
```
feat(dashboard): add agent status real-time monitoring panel
fix(websocket): resolve connection timeout issue
docs(api): update WebSocket event documentation
```
