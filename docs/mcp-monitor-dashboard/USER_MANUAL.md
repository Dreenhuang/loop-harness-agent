# MCP 实时监控看板系统 - 用户操作手册

## 1. 系统简介

MCP 实时监控看板系统提供对 MCP 系统中 16 个 Agent 的实时监控能力，包括：

- **Agent 状态矩阵**: 实时展示所有 Agent 的运行状态（运行中/空闲/错误）
- **操作日志流**: 实时滚动显示系统操作日志
- **项目概览**: 展示项目进度、任务完成度、Token 使用情况
- **系统控制**: 启动/停止/重启 MCP Server

## 2. 访问系统

### 2.1 浏览器访问

打开浏览器，访问：`https://monitor.renrenup.cn`

### 2.2 系统要求

- 现代浏览器：Chrome 90+、Firefox 88+、Edge 90+、Safari 14+
- 推荐分辨率：1920x1080 或更高
- 支持移动端访问（响应式设计）

## 3. 功能说明

### 3.1 Agent 状态矩阵

**位置**: 页面顶部区域

**功能**:
- 显示 16 个 Agent 的实时状态
- 每个 Agent 卡片包含：图标、名称、当前任务、进度条
- 状态颜色：绿色（运行中）、灰色（空闲）、红色（错误）

**操作**:
- 点击 Agent 卡片：查看该 Agent 的详细日志
- 状态自动刷新（每 2 秒）

### 3.2 操作日志流

**位置**: 页面左侧

**功能**:
- 实时显示系统操作日志
- 支持按级别过滤（INFO/WARN/ERROR）
- 支持关键词搜索
- 自动滚动到最新日志

**操作**:
- 使用顶部过滤栏：选择日志级别
- 使用搜索框：输入关键词搜索
- 点击暂停按钮：暂停自动滚动
- 点击清空按钮：清空当前显示

### 3.3 项目概览

**位置**: 页面右侧

**功能**:
- 项目名称和当前阶段
- 总体进度百分比
- 任务统计：总任务数、已完成、失败
- Token 使用情况：已使用/总量
- 门禁状态：Gate 1-4 通过状态

**数据说明**:
- 数据每 2 秒自动更新
- 进度条显示当前阶段完成百分比

### 3.4 系统控制

**位置**: 页面右上角

**功能**:
- 启动 MCP Server
- 停止 MCP Server
- 重启 MCP Server
- 查看系统运行状态

**操作**:
1. 点击"系统控制"按钮
2. 选择操作（启动/停止/重启）
3. 确认操作
4. 查看操作结果

**注意事项**:
- 停止 MCP Server 会导致所有 Agent 停止工作
- 重启操作会先停止再启动
- 操作需要管理员权限

## 4. WebSocket 连接

### 4.1 连接状态

系统使用 WebSocket 实现实时数据推送：

- **连接成功**: 页面右上角显示绿色圆点
- **连接断开**: 显示红色圆点，自动重连
- **重连中**: 显示黄色圆点

### 4.2 自动重连

- 最大重连次数：10 次
- 重连策略：指数退避（1s, 2s, 4s, 8s...最大 30s）
- 超过最大重连次数后，需手动刷新页面

## 5. 常见问题

### Q1: 页面显示"连接断开"怎么办？

**A**: 
1. 检查网络连接
2. 等待自动重连（最多 10 次）
3. 如果仍无法连接，刷新页面（F5）
4. 联系管理员检查后端服务

### Q2: Agent 状态长时间不更新？

**A**:
1. 检查 WebSocket 连接状态
2. 刷新页面
3. 检查 MCP Server 是否正常运行
4. 查看系统日志排查问题

### Q3: 日志显示不完整？

**A**:
1. 调整日志过滤条件
2. 增加页面大小（分页参数）
3. 检查数据库连接
4. 查看后端日志确认数据采集是否正常

### Q4: 移动端访问体验不佳？

**A**:
1. 系统支持响应式设计，但推荐使用桌面端
2. 移动端可以横屏访问
3. 部分功能（如系统控制）在移动端可能需要滚动查看

## 6. 快捷键

系统支持以下快捷键（需聚焦在页面）：

- `F5` 或 `Ctrl+R`: 刷新页面
- `Ctrl+Shift+R`: 强制刷新（清除缓存）
- `Esc`: 关闭弹窗

## 7. 性能指标

### 7.1 响应时间

- 页面加载时间：< 3 秒
- API 响应时间：< 500ms
- WebSocket 消息延迟：< 100ms

### 7.2 并发能力

- 支持 100 个 WebSocket 并发连接
- 支持 50 个并发 API 请求
- 数据库查询时间：< 200ms

### 7.3 数据更新频率

- Agent 状态：每 2 秒更新
- 操作日志：实时推送
- 项目概览：每 2 秒更新
