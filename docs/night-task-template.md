# Loop Agent · 夜间作业模板（Night Task Template）

> **版本**：v1.0
> **生效日期**：2026-06-15
> **用途**：用户晚上设定任务的标准模板

---

## 一、标准夜间作业指令模板

### 1.1 最简版（3 要素）

```text
用 Loop Agent 模式开发，今晚完成明天早上给我结果。
PRD 路径：docs/xxx-prd.md
时间预算：8 小时
```

### 1.2 完整版（7 要素）

```text
【Loop Agent 夜间作业指令】

1. PRD 路径：docs/todo-app-prd.md
2. 时间预算：8 小时（今晚 23:00 → 明天 07:00）
3. 必做功能：
   - 用户登录（手机号 + 验证码）
   - 列表展示（分页 + 搜索）
   - 详情页（图片 + 描述）
   - 提交订单（含支付）
4. 可选功能（如时间允许）：
   - 消息推送
   - 数据统计
   - 导出 Excel
5. 必做质量：
   - 单元测试覆盖率 ≥ 80%
   - P95 响应时间 ≤ 300ms
   - 4 道门禁全部通过
6. 部署目标：staging 环境（待我醒来确认后上生产）
7. 重点关注：安全（SQL 注入 / XSS / 权限）
```

### 1.3 自定义版（含特殊要求）

```text
【Loop Agent 夜间作业指令 · 自定义】

1. PRD 路径：docs/xxx-prd.md
2. 时间预算：6 小时（必须完成）
3. 必做：[列出 MVP 功能]
4. 可选：[列出 nice-to-have]
5. 必做质量：[SLA 指标]
6. 部署目标：[staging/production]
7. 重点关注：[安全/性能/特定领域]
8. 绝对禁止：[不可做的事，如：换技术栈、删数据、改 PRD]
9. 触发决策时优先：[速度/质量/节省 Token]
10. 异常处理：[降级方案/紧急停止条件]
```

## 二、模板变体

### 2.1 "必须完成"型

```text
今晚必完成，时间不是问题。
PRD：docs/mvp-prd.md
必做：核心 3 个功能
质量：测试通过 + 部署 staging
决策：速度优先
```

### 2.2 "质量优先"型

```text
今晚质量优先，宁可少做也要做好。
PRD：docs/production-prd.md
必做：核心功能 + 4 道门禁全过
质量：P95 ≤ 200ms / 覆盖率 ≥ 90%
决策：质量优先
```

### 2.3 "探索性"型

```text
今晚探索多个方案，醒来告诉我哪个最好。
PRD：docs/exploration-prd.md
范围：3 种技术栈对比 + 2 套架构
输出：决策报告 + 推荐方案
时间：8 小时
```

## 五、最佳实践

### 5.1 Do ✅

- ✅ PRD 越详细越好（验收标准、边界条件、异常处理）
- ✅ 必做/可选功能明确分离
- ✅ 质量要求量化（覆盖率、P95、错误率）
- ✅ 部署目标明确（staging/production）
- ✅ 重点关注项明确（安全/性能/特定领域）
- ✅ 绝对禁止项明确（防止 AI 越界）
- ✅ 时间预算合理（8 小时 = 睡觉场景）

### 5.2 Don't ❌

- ❌ PRD 模糊不清（"做个差不多的"）
- ❌ 没有时间预算（"做完了告诉我"）
- ❌ 必做/可选不分（"都做完"）
- ❌ 没有质量要求（"差不多就行"）
- ❌ 部署目标模糊（"上线吧"）
- ❌ 重点关注空白（AI 自己决定）
- ❌ 绝对禁止空白（AI 可能换技术栈）

## 六、故障排查

### 6.1 早晨没看到完整交付物？

| 症状 | 原因 | 排查 |
|------|------|------|
| 报告说完成 90% | Phase 10 部署需确认 | 检查 pending_decision.json |
| 报告说停在 Phase 5 | 严重异常 | 检查 decision_log.json 的 L4 异常 |
| 报告说部分完成 | 时间耗尽 | 调整时间预算到 12 小时 |
| 早晨 AI 还在跑 | 时间没到点 | 等待或手动 /loop-agent abort |
| 报告丢失 | 黑板损坏 | 从 snapshot 恢复 |

### 6.2 如何继续未完成的任务？

```text
/loop-agent resume
→ 读取 项目进度记录.md
→ 读取 decision_log.json
→ 读取 pending_decision.json
→ 询问：是否继续上次任务？
```
