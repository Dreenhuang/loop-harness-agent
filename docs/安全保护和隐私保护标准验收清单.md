## 一、数据加密标准

### 1.1 传输加密（TLS 1.2+）

**状态**: ⚠️ 部署层面需配置

**现状**:
- 代码层面未强制 HTTPS
- 需通过 Nginx SSL 配置实现传输加密

**部署要求**:
```nginx
ssl_protocols TLSv1.2 TLSv1.3;
ssl_ciphers HIGH:!aNULL:!MD5;
ssl_prefer_server_ciphers on;
add_header Strict-Transport-Security "max-age=31536000" always;
```

**验收标准**:
- [ ] Nginx 配置 TLS 1.2+
- [ ] 强制 HTTPS 重定向
- [ ] HSTS 头已启用

---

### 1.2 存储加密（数据库）

**状态**: ⚠️ 已知风险

**现状**:
- SQLite 数据库明文存储
- 无透明数据加密（TDE）

**风险等级**: P3（低风险）

**优化方案**:
- 迁移到 SQLCipher（加密版 SQLite）
- 或迁移到 PostgreSQL/MySQL（支持 TDE）
- 或加密敏感字段（API Key、Token 等）

**当前缓解措施**:
- 数据库文件权限最小化（仅应用用户可读写）
- 生产环境部署在受保护的服务器目录

---

### 1.3 存储加密（日志）

**状态**: ⚠️ 部分实现

**现状**:
- 日志系统有基础脱敏（顶层 key 匹配）
- 不覆盖嵌套结构和异常堆栈

**风险等级**: P2（中风险）

**优化方案**:
- 实现递归脱敏（覆盖嵌套字典）
- 异常堆栈截断或二次过滤
- URL 参数正则脱敏

---

### 1.4 密钥管理

**状态**: ✅ 通过

**现状**:
- API Key 通过环境变量 `API_KEY` 读取
- 无硬编码密钥

**生产环境要求**:

**验收标准**:

**状态**: ✅ 通过（已实现）

**实现方案**:


**验收标准**:
- [x] 所有非白名单路径需 API Key
- [x] 错误 API Key 返回 401
- [x] 白名单路径无需鉴权

### 2.2 WebSocket 鉴权

**状态**: ✅ 通过（已实现）

**实现方案**:
- WebSocket 连接时校验 `api_key` query 参数或 header
- 校验失败返回 close code 1008（Policy Violation）

**验证命令**:
```javascript
// 无 API Key 连接应被拒绝
const ws = new WebSocket("ws://localhost:8000/ws");
// 期望：连接被关闭，code 1008

// 携带 API Key 连接应成功
const ws = new WebSocket("ws://localhost:8000/ws?api_key=your-key");
// 期望：连接成功
```

**验收标准**:
- [x] WebSocket 连接需 API Key
- [x] 错误 API Key 返回 1008
- [x] 正确 API Key 连接成功


### 4.2 日志脱敏规则

**状态**: ⚠️ 简单 key 匹配，不覆盖嵌套/堆栈

**现状**:
- `_sanitize_args()` 仅对字典顶层 key 做精确匹配
- 嵌套字典中的敏感字段不会被脱敏
- 异常堆栈可能泄露 token/password

**风险等级**: P2（中风险）

**优化方案**:
```python
def _sanitize_recursive(obj, sensitive_keys, depth=0):
    if depth > 10:  # 防止无限递归
        return obj
    if isinstance(obj, dict):
        return {k: _sanitize_recursive(v, sensitive_keys, depth+1) 
                for k, v in obj.items()}
    elif isinstance(obj, list):
        return [_sanitize_recursive(item, sensitive_keys, depth+1) 
                for item in obj]
    else:
        return obj
```
