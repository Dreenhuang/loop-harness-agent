## 一、验收背景

### 1.1 问题起源

用户报告MCP架构在调用时出现异常行为——**所有接口调用均生成.md文件而非执行实际开发功能**。

### 1.2 根因分析

经过深入排查，确认根本原因：

| 层级 | 原始设计 | 实际问题 |
|------|----------|----------|
| **架构设计** | "元调度器"模式 | `spawn_agent()` 仅返回提示信息，不执行任务 |
| **工具实现** | 12个工具全部为只读/元操作 | 无实际代码生成能力 |
| **响应格式** | 统一返回元数据 | 缺少 executed/error/hint_only 模式区分 |
| **文件操作** | 仅 `save_blackboard` 可写文件 | 且只能写 .md 格式 |

### 1.3 解决方案

**方案A：重构MCP为"智能执行器"**（已实施并完成）

### 5.2 数据流验证

**正常流程（以spawn_agent backend为例）**:

```
客户端请求 → server.py → dispatcher.py
                         ↓
              ┌──────────────────────────┐
              │ 1. orchestrator.spawn_agent()│ ← 状态更新
              │ 2. executors.execute_backend()│ ← 代码生成
              │ 3. 合并结果                   │ ← {orch_result + exec_result}
              └──────────────────────────┘
                         ↓
              返回给客户端 {
                status: "executed",
                mode: "executed",
                agent: "backend",
                files_created: ["src/api/users.ts", ...],
                output: "✅ 后端任务完成：生成 2 个文件"
              }
```

```

                                 ↓
                    返回 {
                      status: "error",
                      error: "未知角色: invalid_role"
                    }
```
