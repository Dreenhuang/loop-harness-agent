### 4.3 闭环规则

1. **Blocker**：必须修复并复核通过后才能合并
2. **Major**：必须修复，可在同一 PR 中修复
3. **Minor**：建议修复，可创建 follow-up Issue
4. **Suggestion**：可选，记录但不阻塞
5. 所有修复必须由 @Code-Reviewer 复核确认
