## 3. 风险预案

<!-- 填写指引：识别开发过程中的潜在风险，制定预防和应对措施。 -->

### 3.1 风险登记

| 风险 ID | 风险描述 | 概率 | 影响 | 风险等级 | 预防措施 | 应对措施 |
|---------|----------|------|------|----------|----------|----------|
| R-001 | 第三方 API 不可用 | 中 | 高 | 🔴 高 | 提前测试 API 可用性 | Mock 数据降级 |
| R-002 | 技术栈兼容问题 | 低 | 高 | 🟡 中 | 提前做技术验证 | 回退到兼容方案 |
| R-003 | 需求变更 | 中 | 中 | 🟡 中 | 需求冻结协议 | 评估影响后决策 |
| R-004 | 性能不达标 | 低 | 中 | 🟢 低 | 早期性能基准测试 | 优化或降级功能 |
| R-005 | 关键路径阻塞 | 低 | 高 | 🟡 中 | 并行策略预留缓冲 | 切换到非阻塞任务 |

### 3.2 应急流程

```
风险发生
  ├── 评估影响范围和程度
  ├── 通知 @Orchestrator
  ├── 选择应对策略
  │   ├── 规避：修改计划绕过风险
  │   ├── 转移：交由其他角色处理
  │   ├── 缓解：降低风险影响
  │   └── 接受：记录并继续
  ├── 执行应对措施
  └── 更新黑板 + 风险登记
```
