## 一、PRD 摘要

### 1.1 核心定位
将 MiniMax 配额监控工具从**单厂商**扩展到**国内 8 家 Coding Plan + 用户插件无限扩展**，重点不是 API 调用量，而是**订阅计划/Coding Plan 剩余额度**。

### 1.2 关键交付物
- 8 家内置厂商插件（MiniMax / 智谱 / 腾讯云 / 阿里云 / 火山 / Kimi / 阶跃 / DeepSeek）
- 插件化扩展机制（JSON 描述符 + 5 分钟用户自定义）
- 完整 v1.0.6 向后兼容
- Mock 测试数据集（无真实账号验证）

### 1.3 验收维度（22 项）
- 🅰️ 功能完整性 6 / 🅱️ 兼容性 3 / 🅲️ 性能 4 / 🅳️ 稳定性 3 / 🅴️ 可扩展性 2 / 🅵️ 插件引擎 4

## 三、子任务拆分（18d × 16 角色）

### Day 1-2: Phase 1 需求 + 验收
- **requirements**：基于 v2.0 文档细化 22 项验收为可测用例
- **product-manager**：签字 G1
- **orchestrator**：协调 + 记录黑板

### Day 3-4: Phase 2 架构 + Schema
- **architect**：CodingPlanProvider trait / CodingPlanUsage 数据模型 / Plugin JSON Schema 冻结
- **backend**：Rust 项目骨架 + Cargo.toml 依赖
- **product-manager**：签字 G2

### Day 5: Phase 3 UX/UI 设计
- **ux-researcher**：用户添加自定义厂商旅程
- **ui-designer**：设置面板 + 插件向导 + 详情面板设计
- **product-manager**：签字 G3

### Day 6-7: Phase 4 前端基础
- **frontend**：Vue 3 组件库 + Tauri IPC 客户端 + 类型生成

### Day 8-11: Phase 5 后端插件
- **backend**：8 家内置插件依次实现（每厂商 0.5-1d）
  - MiniMax 迁移 / 智谱 / 腾讯云 / 阿里云 / 火山 / Kimi / 阶跃 / DeepSeek
- **code-reviewer**：每厂商实现完成触发 L1 自审 + L2 业务审

### Day 12-13: Phase 6 前端集成
- **frontend**：插件向导 UI + 设置面板 + 详情面板集成
- **backend**：IPC 接口对接

### Day 14-15: Phase 7 测试 + 性能
- **tester**：L2 fixture + L3 mock server + L4 文档对齐
- **performance**：8 厂商并行 < 5s + 72h 长跑
- **code-reviewer**：L3 终审

### Day 16-17: Phase 8 BUG + 文档
- **bug-fix**：修复所有测试发现 BUG
- **knowledge-curator**：6 段式教程沉淀
- **documenter**：用户使用手册 + Release Notes

### Day 17-18: Phase 9 验收
- **code-reviewer**：L1 / L2 / L3 终审签字
- **product-manager**：业务验收
- **final-reviewer**：最终签字 G4

### Day 18: Phase 10 发布
- **final-reviewer**：合并 main 签字 G5
- **devops**：Tauri 打包 + .exe 构建
- **documenter**：Release Notes 发布
- **orchestrator**：黑板归档
